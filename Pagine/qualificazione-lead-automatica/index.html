Qualificazione Lead Automatica

Per chi riceve lead da Meta Ads e non riesce a rispondere abbastanza velocemente

[ definizione tecnica ]

Come funziona la qualificazione lead automatica con AI

Il sistema applica una versione conversazionale del framework BANT (Budget, Authority, Need, Timeline) adattata al contesto specifico. L'agente AI non pone le domande in sequenza rigida come un form: introduce il budget in modo naturale, verifica chi è il decision-maker, esplora il problema con domande di discovery, valuta l'urgenza. L'ordine varia in base al flusso della conversazione, esattamente come farebbe un buon commerciale.

Il flusso tecnico in 7 fasi: (1) il lead entra da Meta Ads, QR code o opt-in form → (2) il gateway WhatsApp riceve il messaggio via webhook e lo passa a n8n → (3) n8n verifica se esiste una sessione attiva per quel numero → (4) l'LLM elabora il messaggio con il system prompt configurato → (5) quando l'agente ha raccolto abbastanza informazioni, chiama il tool finalize_conversation con outcome, reason e summary → (6) azioni downstream: qualified → link calendario; nurture → contenuto; rejected → chiusura professionale → (7) tutto viene loggato automaticamente nel CRM.

Il Qualification Intelligence Brief è il documento che definisce il sistema prima del go-live: profilo del cliente ideale (ICP), criteri di qualifica specifici (budget minimo, urgenza, fit), domande sequenziali, logica di decisione per outcome, tono e stile della comunicazione, istruzioni di fallback per domande fuori perimetro. Senza questo brief, il sistema qualifica secondo parametri generici. Con questo brief, qualifica secondo i tuoi criteri esatti.

Cosa qualifica il sistema (e dove si ferma)

  • ✓ Risponde entro 60 secondi dall'opt-in Meta Ads — prima che il lead si raffreddi
  • ✓ Pone domande di qualifica in linguaggio naturale, non con form statici o menu a bottoni
  • ✓ Adatta la sequenza delle domande in base alle risposte precedenti — come un commerciale reale
  • ✓ Valuta budget, urgenza, autorità decisionale e fit con il profilo cliente ideale (framework BANT)
  • ✓ Gestisce le obiezioni più frequenti (prezzo, tempo, "ci penso") con risposte configurate
  • ✓ Produce output strutturato per ogni lead: outcome, reason, summary — direttamente nel CRM
  • ✓ Invia il link calendario solo ai lead qualificati — gli altri ricevono una risposta educativa
  • ✓ Loga ogni conversazione in Google Sheets, HubSpot o Airtable senza inserimento manuale
  • ✓ Notifica il team via Slack o email per ogni lead qualificato, con trascrizione completa
  • ✓ Opera 24/7 — qualifica anche alle 23:00 di domenica e durante i lanci con 500 lead/giorno

// cosa non fa

  • ✗ Non sostituisce la discovery call: prepara il lead, non chiude il contratto
  • ✗ Non qualifica correttamente senza un Qualification Intelligence Brief ben configurato — richiede 20-40 minuti di setup iniziale
  • ✗ Non gestisce contenuto multimediale (immagini, documenti) senza processing aggiuntivo
  • ✗ Non funziona su lead senza opt-in esplicito — il GDPR richiede consenso per il trattamento dei dati

[ implementazione ]

Come si implementa la qualificazione lead automatica

Il setup base con il Blueprint richiede 85-130 minuti divisi in 4 fasi. Il setup avanzato con Evolution API self-hosted aggiunge 60-90 minuti di provisioning VPS.

01
Fase 0 — Qualification Intelligence Brief (20-40 min)
Definire per iscritto: chi è il tuo cliente ideale (ICP), quale problema risolvi, qual è il budget minimo che qualifica un lead, quale urgenza è richiesta, chi è il decision-maker nel processo di acquisto. Queste informazioni diventano il system prompt dell'agente. Più specifico è il brief, più precisa è la qualifica.
→ output: brief compilato con ICP, criteri di qualifica, tono e istruzioni di fallback
02
Fase 1 — Setup Dify + personalizzazione agente (30-40 min)
Creare account Dify su dify.ai → importare configurazione agent precompilata dal Blueprint → personalizzare il system prompt con i tuoi criteri specifici → configurare il modello LLM (Claude Sonnet 4 o GPT-4o come principale, Claude Haiku come fallback economico) → creare il Custom Tool finalize_conversation → testare nella console Dify prima di connettere al gateway.
→ output: agente Dify configurato e testato con i tuoi criteri di qualifica
03
Fase 2 — Import n8n bridge + configurazione webhook (15-20 min)
In n8n: importare il workflow bridge dal Blueprint → configurare il nodo Webhook che riceverà i messaggi da 2Chat → configurare HTTP Request verso Dify API con le credenziali → configurare HTTP Request verso 2Chat per inviare risposte → aggiungere il filtro fromMe=true per evitare loop di messaggi → attivare il workflow.
→ output: bridge n8n ↔ Dify ↔ 2Chat funzionante, filtro loop attivo
04
Fase 3 — CRM sync + speed-to-lead + go-live (15-20 min)
In n8n: importare il workflow CRM sync → configurare Google Sheets con OAuth (o HubSpot/Airtable se preferito) → collegare il webhook del CRM sync al tool finalize_conversation di Dify → configurare il trigger speed-to-lead (webhook dal form/landing page → n8n invia primo messaggio personalizzato entro 60 secondi) → test end-to-end con numero di test → attivare in produzione.
→ output: sistema live: lead qualificati nel CRM, speed-to-lead < 60s attivo

[ domande frequenti ]

Domande frequenti sulla qualificazione lead automatica

Come fa l'agente AI a capire se un lead è qualificato o no?
Il sistema applica i criteri che tu definisci nel Qualification Intelligence Brief: budget minimo, urgenza richiesta, fit con il profilo cliente ideale, autorità decisionale. L'LLM interpreta le risposte in linguaggio naturale e valuta se i criteri sono soddisfatti. Quando ha abbastanza informazioni, chiama il tool finalize_conversation con l'esito (qualified / nurture / rejected) e una motivazione testuale. Non è un sistema a punteggio rigido: è un ragionamento contestuale.
Cosa succede ai lead che non superano la qualifica?
Dipende dall'esito configurato. I lead da nurturare ricevono un contenuto pertinente (link a un articolo, a una risorsa gratuita) e vengono inseriti in una sequenza di follow-up automatica. I lead non adatti ricevono una risposta professionale di chiusura con porta aperta — mai un rifiuto secco. In entrambi i casi, i dati vengono loggati nel CRM per analisi future.
Quanti lead al mese può gestire il sistema?
Con architettura cloud (Dify + n8n Cloud + 2Chat): la scalabilità è praticamente illimitata per volume di conversazioni. Il limite pratico è il gateway WhatsApp: con 2Chat (non ufficiale), oltre 200-300 conversazioni/giorno per numero aumenta il rischio di ban. Per volumi enterprise si usa l'API ufficiale Meta tramite BSP certificato.
Il sistema funziona anche per chi non usa Meta Ads?
Sì. Il trigger può essere qualsiasi canale: link WhatsApp in bio Instagram, QR code su materiale fisico, firma email, sito web con widget WhatsApp, oppure messaggio spontaneo al numero aziendale. La qualificazione funziona indipendentemente dal canale di entrata. Meta Ads è il caso d'uso più comune perché genera alti volumi di lead che richiedono risposta rapida.
Quanto è configurabile il sistema di qualifica?
Completamente. Il Qualification Intelligence Brief definisce tutto: le domande specifiche, l'ordine, il tono, i criteri di scoring, le risposte alle obiezioni più frequenti, le istruzioni di escalation. Due business con lo stesso stack tecnico possono avere sistemi di qualifica radicalmente diversi. La personalizzazione del brief richiede 20-40 minuti e può essere aggiornata dopo ogni ciclo di campagna in base ai pattern reali delle conversazioni.
